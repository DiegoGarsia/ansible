# Ansible Lab

Проект содержит набор Ansible playbook'ов и ролей для автоматизированной настройки серверов.

Основная цель — хранить инфраструктуру в виде кода (Infrastructure as Code), чтобы новые серверы могли быть подготовлены и настроены за несколько минут.

---

# Возможности

На данный момент проект позволяет:

* выполнить базовую настройку новых серверов (Bootstrap);
* установить и настроить Nginx;
* развернуть и настроить Bind9 DNS;
* централизованно управлять конфигурацией через Ansible Inventory;
* использовать отдельные переменные для групп и отдельных узлов.

---

# Структура проекта

```text
ansible/
├── ansible.cfg                # Конфигурация Ansible
├── inventory/
│   ├── hosts.ini              # Inventory
│   ├── group_vars/            # Общие переменные
│   └── host_vars/             # Переменные отдельных узлов
├── playbooks/                 # Основные playbook'и
└── roles/                     # Ansible роли
```

---

# Требования

Для работы потребуется:

* Linux / macOS (или WSL);
* Ansible;
* SSH-доступ к управляемым серверам;
* установленный OpenSSH.

Проверить установленную версию можно командой:

```bash
ansible --version
```

# Подготовка проекта

Склонируйте репозиторий:

```bash
git clone <repository-url>
cd ansible
```

Перед первым запуском необходимо добавить свой публичный SSH-ключ.

В каталоге

```text
roles/bootstrap/files/
```

находится шаблон файла:

```text
access_key.pub.example
```

Создайте рабочий файл:

```bash
cp roles/bootstrap/files/access_key.pub.example \
   roles/bootstrap/files/access_key.pub
```

Откройте `access_key.pub` и замените содержимое своим публичным SSH-ключом.

> Рабочий файл не добавляется в Git и используется только локально.

---

# Настройка Inventory

Перед запуском playbook необходимо указать целевые серверы.

Файл inventory расположен здесь:

```text
inventory/hosts.ini
```

Пример:

```ini
[web]
192.168.10.10

[dns]
192.168.10.20
```

При необходимости можно использовать собственный inventory:

```bash
ansible-playbook -i inventory/hosts.ini playbooks/bootstrap.yml
```

---

# Запуск Playbook

Bootstrap нового сервера:

```bash
ansible-playbook playbooks/bootstrap.yml
```

Настройка Nginx:

```bash
ansible-playbook playbooks/nginx.yml
```

Настройка Bind9:

```bash
ansible-playbook playbooks/bind.yml
```

---

# Переменные

Общие переменные находятся в:

```text
inventory/group_vars/
```

Переменные отдельных хостов:

```text
inventory/host_vars/
```

Изменение параметров рекомендуется выполнять именно через эти каталоги, не изменяя сами роли.

---

# Добавление новых ролей

Новые роли рекомендуется создавать стандартной командой Ansible:

```bash
ansible-galaxy role init <role_name>
```

После этого роль подключается в соответствующем playbook.

---

# Безопасность

В репозиторий **не включаются**:

* персональные SSH-ключи;
* приватные ключи;
* пароли;
* секреты и токены.

Все подобные данные должны храниться локально или в специализированном хранилище секретов.
