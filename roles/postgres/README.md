# postgres role

## Описание

Роль предназначена для установки и первоначальной настройки PostgreSQL.

После выполнения роли автоматически:

- устанавливается PostgreSQL;
- запускается сервис PostgreSQL;
- создается пользователь базы данных;
- создается тестовая база данных;
- импортируются тестовые данные;
- обеспечивается идемпотентность повторного запуска.

---

## Используемые компоненты

- PostgreSQL
- community.postgresql
- psql

---

## Структура роли

```
postgres/
├── defaults/
├── files/
├── tasks/
├── handlers/
└── README.md
```

---

## Используемые переменные

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| postgres_db | testdb | Имя базы данных |
| postgres_user | testuser | Пользователь PostgreSQL |
| postgres_password | strongpassword | Пароль пользователя |
| postgres_packages | postgresql, postgresql-contrib | Устанавливаемые пакеты |

---

## Выполняемые действия

### Установка PostgreSQL

Устанавливаются необходимые пакеты:

- PostgreSQL
- PostgreSQL Contrib

После установки сервис автоматически запускается.

---

### Создание пользователя

Создается пользователь PostgreSQL.

При повторном запуске playbook пользователь повторно не создается.

---

### Создание базы данных

Создается база данных и назначается владелец.

---

### Импорт тестовых данных

В базу импортируется SQL-файл:

```
roles/postgres/files/sql_foundation.sql
```

Перед импортом выполняется проверка наличия таблицы:

```
superheroes
```

Если таблица существует, импорт пропускается.

Это делает роль полностью идемпотентной.

---

## Проверка

Подключиться к базе:

```bash
sudo -u postgres psql -d testdb
```

Посмотреть таблицы:

```sql
\dt
```

Проверить количество записей:

```sql
SELECT COUNT(*) FROM superheroes;
```

---

## Особенности

- Полностью идемпотентна.
- Использует FQDN (`ansible.builtin.*`).
- Импорт SQL выполняется только один раз.
- Подходит для повторного запуска без изменения существующих данных.
