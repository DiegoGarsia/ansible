# Bootstrap Role

## Описание

Роль **bootstrap** предназначена для первичной подготовки новых серверов после их создания Terraform или вручную.

Роль выполняется **один раз** сразу после развертывания операционной системы и подготавливает сервер к дальнейшему управлению через Ansible.

После выполнения роли сервер становится готовым для применения остальных ролей проекта (`bind`, `nginx`, `docker`, `postgres` и др.).

---

## Выполняемые задачи

Роль выполняет следующие действия:

1. Обновляет систему.
2. Устанавливает базовый набор пакетов.
3. Настраивает системную временную зону.
4. Создает административного пользователя.
5. Добавляет SSH-ключ администратора.
6. Добавляет пользователя в группу sudo (Debian) или wheel (RedHat).
7. Настраивает sudo без запроса пароля.
8. Выполняет базовое усиление безопасности SSH:
   - запрещает вход по паролю;
   - запрещает вход пользователю root;
   - разрешает вход только пользователю администратора;
   - оставляет только авторизацию по SSH-ключам.
9. Устанавливает и включает QEMU Guest Agent (только для виртуальных машин KVM).

## Особенности

### Обновление системы

В зависимости от семейства ОС используется соответствующий менеджер пакетов.

- Debian/Ubuntu — `apt`
- RedHat/CentOS — `dnf`

---

### Базовые пакеты

Устанавливаются необходимые утилиты для администрирования сервера.

Например:

- sudo
- vim
- curl
- wget
- git
- bash-completion
- ca-certificates
- qemu-guest-agent (только KVM)

---

### Создание администратора

Роль создает пользователя:

```
diego
```

(или пользователя, указанного в переменной `bootstrap_user`)

Для пользователя:

- создается домашняя директория;
- устанавливается оболочка;
- копируется публичный SSH-ключ;
- пароль блокируется;
- пользователь получает права администратора.

---

### Настройка SSH

Роль изменяет `/etc/ssh/sshd_config`.

После выполнения:

- PasswordAuthentication → `no`
- ChallengeResponseAuthentication → `no`
- PermitRootLogin → `no`
- PubkeyAuthentication → `yes`
- AllowUsers → `<bootstrap_user>`

После изменения конфигурации автоматически выполняется перезапуск службы SSH.

---

### QEMU Guest Agent

Устанавливается **только** на виртуальные машины KVM.

Контейнеры LXC автоматически пропускают этот этап.

---

## Переменные

Основные переменные находятся в:

```
defaults/main.yml
```

Например:

```yaml
bootstrap_user: diego
bootstrap_shell: /bin/bash

bootstrap_timezone: Europe/Moscow

bootstrap_public_key: files/access_key.pub
```

---

## Использование

Пример playbook:

```yaml
---
- name: Bootstrap new servers
  hosts: managed
  become: true

  roles:
    - bootstrap
```

Запуск:

```bash
ansible-playbook playbooks/bootstrap.yml
```

---

## Жизненный цикл

Роль рассчитана на **первичный запуск** после создания сервера.

Типичный сценарий выглядит следующим образом:

```
Terraform
        │
        ▼
Создание сервера
        │
        ▼
Временный пользователь (ubuntu)
        │
        ▼
Bootstrap
        │
        ▼
Создание администратора (diego)
        │
        ▼
Настройка SSH
        │
        ▼
Запрет входа root
        │
        ▼
Дальнейшее управление через пользователя diego
```

После успешного выполнения роли рекомендуется изменить `ansible_user` на постоянного администратора (`diego`) и использовать его для всех последующих запусков Ansible.

## Поддерживаемые операционные системы

- Ubuntu 24.04 LTS
- CentOS Stream 9

---

## Требования

- SSH-доступ к серверу.
- Наличие публичного SSH-ключа.
- Поддержка `sudo`.
- Python установлен на целевой машине.
