# backup role

## Описание

Роль предназначена для автоматической настройки резервного копирования PostgreSQL.

После выполнения роли автоматически:

- создается директория для хранения резервных копий;
- разворачивается скрипт резервного копирования;
- создаются systemd unit и timer;
- включается автоматический ежедневный запуск резервного копирования;
- хранится только заданное количество последних резервных копий.

---

## Используемые компоненты

- PostgreSQL
- pg_dump
- systemd service
- systemd timer

---

## Используемые переменные

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| backup_database | testdb | Имя резервируемой базы данных |
| backup_directory | /var/backups/pg | Каталог хранения резервных копий |
| backup_script | /usr/local/bin/pg-backup.sh | Путь до скрипта резервного копирования |
| backup_retention | 7 | Количество сохраняемых резервных копий |
| backup_hour | 3 | Час запуска |
| backup_minute | 0 | Минута запуска |

---

## Принцип работы

Роль устанавливает:

- systemd service

```
pg-backup.service
```

- systemd timer

```
pg-backup.timer
```

Каждый день в указанное время выполняется:

```
pg_dump --clean --if-exists --create
```

После успешного создания резервной копии автоматически удаляются старые файлы, оставляя только последние 7 копий.

---

## Проверка

Проверить состояние timer:

```bash
systemctl status pg-backup.timer
```

Просмотреть журнал:

```bash
journalctl -u pg-backup.service
```

Запустить резервное копирование вручную:

```bash
systemctl start pg-backup.service
```

Проверить созданные резервные копии:

```bash
ls -lh /var/backups/pg
```

## Особенности

- Полностью идемпотентна.
- Использует только встроенные модули Ansible (`ansible.builtin.*`).
- Не использует cron.
- Логирование осуществляется через systemd-journald.
- Хранит последние N резервных копий.
